creating static template

Nevertheless that we are doing simple application we will start from design and prototyping. That is standard procedure in real life. We found it extremely useful as that shortens development loop. Web designer and Developer work on same HTML&CSS sources.

So, we will create prototype of the application in pure HTML. We will create following pages which you will find in most applications exposing content:

login page
with just username and password fields to keep our project simple
search page
which will let you provide search query and initiate search function
search result
page obviously to present list of found entires
document view
page to complete functional scope of our application

We will not devise CSS stylesheets from scratch. We will use Twitter Bootstrap library. Visit Bootstrap tutorial pages for information on base CSS and components provided in the library. For page flow we will use jQuery .

Have a look at index.html code presented below. Please, press button on right top menu bar on code view area. It will cause collapse of all groups in index.html. As you can see now, we created a single page which encompass functionality for all pages that will be created in our tutorial.

Press "preview" tab on top of code view area to switch it to preview mode. You can play a little bit with that site to familiarize with main page flow of the application. Press "Login" button to get to main application page. Press "Search" button to see how search result page should look like. Click at document link to see document view page.

Notice that you are working with prototype with no logic behind. You will be shown all pages without any need to provide data.

Now we have a prototype of our future application. We know how it should look like, how the page flows are organized. We will use that HTML page as starting point for rest of steps in this tutorial. You may copy&paste that code to run it in you environment or download full version of web application.
