Up until now tutorial application was only a prototype. You will add logic to your application. It will use back-end system functionality exposed by Velvet interface. Two new models for searching and for document view will be used. Those are implemented in SearchModel and DocumentModel and are defined in configuration under $search and $document aliases accordingly. Both are defined in app.js file. 


        

    

    

        

            We use $search and $document just by convention we found clear. You may choose as aliases any other names.
 
			You may also define more then one model objects of same kind e.g. $document1 and $document2 to let simultaneous access to two documents. 
        

    

	

		Examine res.html and form.html code snippets to see how $search and $document models are used. You will noticed that in res.html page two local variables have been defined: 
		

        
pageNo

        
it represents number of current result page to be shown

        
perPage

        
it is number of result items presented on one page

		
 
		Both are required by search paging functionality which allows presentation only certain number of result items. Assuming that pageNo = 2 and perPage = 50, application will present result items from 51st to 100th.
	

	

		Notice how pageNo and perPage are initialized (line 3.) and how navigation buttons use them  (line 24.).
