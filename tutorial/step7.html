application parametrization

Up until now the only parametrization of test application has been done via application configuration. But in real life you will use many other parameter providers. Perhaps some data will be stored at client site (browser local store, user session). Some preferences may be read from user profile. And of course you will use CRAs.

1. Data Providers

Lynx gives an easy way to get or set parameter values from those sources. you possibility to access application parameters from following sources:

SessionDataProvider
manages properties stored in browser cache; keep in mind that it may be cleared by user action (e.g. with Ctrl+F5)
PreferencesDataProvider
exposes preferences stored in GARE
CraDataProvider
provides read-only access to CRAs defined in Jaguar
StoreDataProvider
exposes local storage of your application as defined by HTML5 Web Storage specification
StaticDataProvider
provides access to static properties considered as constants

We will use PreferenceDataProvider in our tutorial. First, we will add configuration of that data provider to our application. We will do that in dedicated js file. 


        

        

            

        

    

	
2. Application Properties

	

		You may use many data providers data providers you need to use in your application. Each of data providers configured will be ready for your application. You will not access them directly but via $properties object. It is implemented by lynx.core.AppProperties object. Any operation on $properties scans all data providers one by one.

    

        To access application properties use following functions of $properties object:
		

        
get(name)

        

			Returns value of property name from first data provider that has property defined. In case property is not found at any of defined data providers an exception is thrown.
		

        
set(name, value)

        

			Sets new value for property name. It changes value on first data provider which exposed that property. An exception is thrown if property is not found if it is exposed by read-only data provider.
