Getting started

Prerequisites

  1. Windows: At now the project toolset depends on Windows. You can work on Linux or Mac but the custom configuration will be needed.
  2. Java JRE: Please install the Java JRE. Java is required by build tool for javascript compiling and obfuscation.
  3. Chrome: web browser used by default for e2e tests. Please install following plugins:
    1. JetBrains IDE Support
    2. AngularJS Batarang plugin
    3. Clear Cache
    4. CSS Reloader
  4. OS environment variable: CHROME_BIN pointing to installation folder of Chrome. On windows platform that means setting it to: “c:\Program Files\Google\Chrome\Application\chrome.exe”. Change in environment will take effect after reopening WebStorm (if more then one WebStorm window are open all of them should be closed).
    It’s related to bug in Karma/testacular launchers library for Chrome (see Karma/lib/launchers/Chrome.js). Currently launch command for win32 is defined as follows: "win32: process.env.LOCALAPPDATA + '\\Google\\Chrome\\Application\\chrome.exe'" so it’s based on %LOCALAPPDATA% folder Instead of %PROGRAMFILES% environment variable.

IDE setup

  1. WebStorm IDE v6.0 : Although you can write code in any IDE and use tools by command line, Lynx project is prepared for WebStorm/Idea IDEs.
  2. WebStorm TFS Plugin: Please install TFS plugin (Team Foundation Server – version control server from MS stack) for WebStorm.
    In newly installed WebStorm with no projects opened please click "Configure" and then "Plugins" links on "Quick Start" section -> "Plugins" window will appear; on "Plugins" window type "tfs" in search field (available at top right corner of the window), navigate to file>settings>plugins>browse repository>type "tfs" in search field
  3. Angular Plugin: It adds autocompletion to WebStorm/Idea IDE of AngularJS ng attributes. To install:
    • Navigate to File->Settings->Plugins
    • Click "Browse Repositories"
    • Select "AngularJS"
    • Double-click (or right-click) and, when prompted, choose "Yes"
    • Restart WebStorm
  4. Add in WebStorm settings File Types section *.cshtml extension as valid for HTML

Check-out project

Get project from TFS version control server. You need to have access to the GPO TFS http://tfsgpo.wolterskluwer.com:8080/tfs/gpo please check out Lynx/Dev/Dev-Main/Lynx.
You can do this in WebStorm or in VisualStudio.

For Linux/OSX set execute permission on Base/Tools/JSDoc3/jsdoc and on Base/Tools/NodeJs/node-linux-<arch> for Linux or Base/Tools/NodeJs/node-darwin-<arch> for OSX

Build

Run configuration "Web Server". Do to that choose configuration from drop-down available on tool bar and then click green triangle visible right to configuration drop-down.
Alternatively, press Shift-Alt-F10 and choose "Web Server" configuration from popup window.

Run configuration "Dist". It will build Lynx library (see Base/Core/Client/dist).
Then run "Build LynxSite" configuration. That task depends on files generated by "Dist" configuration. As a final result you will get LynxSite build at Base/Apps/LynxSite/LynxSiteWeb/dist/app. Just open Chrome browser and go to that folder and open index.html file.
