directives & filters documenting

In order to differentiate between directives, filters and regular functions one should use @lynxdoc JSDoc tag. This tag should have value "directive" for directives, and "filter" for filter. It's essential for proper documentation generation that comment also contains @name and @memberOf tags. @memberOf tag contains namespace where respective directive or filter is defined. See example below: 

    /**
    * @directive lx-blur
    * @memberOf lynx.directives
    * @lynxdoc directive
    * @description
    *
    * Factory function of <br/><br/>
    * <h3>lx-blur</h3>
    * <span class="type-signature">attribute directive</span>
    * <br/><br/>
    *     <i>
    *         <any <b>lx-blur</b>="{expression}" />
    *     </i>
    * <br/><br/>
    *
    * This is directive for handling blur event on an element.
    *
    * <p>
    *     <b>Parameters:</b>
    *     <ul>
    *         <li>{expression} - expression will be evaluated when blur event occurs on element</li>
    *     </ul>
    * </p>
    *
    * @example
    * <input type="text" <b>lx-blur</b>="out = 'blur on element'" />
    * {{out}}
    */

Comment shouldn't contain tags changing doclet kind, for example @method, @function, @namespace, @member
