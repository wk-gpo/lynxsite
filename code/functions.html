functions documenting

Private / public functions

Functions are annotated with @private or @public / @expose similarly to members. 

           /**
            * @private
            * ...
            */
            lynx.core.State.prototype.internalGet = function() {...};

           /**
            * @public
            * @expose
            * ...
            */
            lynx.core.State.prototype.get = function() {...};

Function parameters & returned value

In case of functions you need to use two other annotations:

  • @arg which defines parameters of function and may be missing if function does not require any parameters. Do not use JSDoc's @param tag as it iterfers with Closure Compiler
  • @return to specify return type of function; you should always define that annotation
See example below: 
    /**
     * @public
     * @expose
     * @arg {Array.<string>} providers  list of data provider names in order you need to configure
     * @return {void}
     */
    lynx.config.AppProperties.prototype.setProvidersPriority = function (providers) { ... }

Documenting example use of function

You may also use also use @example annotation if you need to show some example of function use. All lines between @example and any other annotation will be used as code snippet in generated API doc. 

    /**
     * @public
     * @expose
     * @arg {Array.<string>} providers  list of data provider names in order you need to configure
     * @return {void}
     * @example
     * $properties.setProvidersPriority([
     *                  "$storeDataProvider",
     *                  "$sessionDataProvider",
     *                  "$craDataProvider",
     *                  "$preferencesDataProvider",
     *                  "$staticDataProvider"
     *                  ]);
     */
    lynx.config.AppProperties.prototype.setProvidersPriority = function (providers) { ... }

Suppressing public function in API Doc

In some cases although function is public it should not be present in API documentation. For that purpose @internal annotation has been provided. It makes the function available for intermodule calls but suppresses generation of API doc. 

    /**
     * @internal
     * @expose
     * @arg {lynx.config.BaseDataProvider} configProvider
     * @return {void}
     */
    lynx.config.AppProperties.prototype.register = function (configProvider) { ... }
Function register would be exposed to other modules but will not be present in API doc.
