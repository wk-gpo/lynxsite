links in documentation

@lynxsitelink tag

Normal JSDoc mechanism for putting links to other resources in "See" section (@see and @link) will not work for documents in lynxsite. It assumes that link target is either another generated page or external URL

Another mechanism had to be put in place that knows LynxSite url structure. All URL's in LynxSite are in the form ...docs.html#<actual location>. For eg. <actual location> for this page is "/codedoc/links". When one needs to put link to LynxSite document in generated docs, one should put comment tag in the form @lynxsitelink <link label>|<actual location> See example below, it will create link labeled "Authorisation" leading to "/func/authorization" page.
