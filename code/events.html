events documenting

Events should be commented in there own ("floating") comments containing @event tag followed by event name in format <class_name>#<event_name>. Class name should be fully qualified, including namespace. Keeping this format is crucial for proper documentation generation.

Functions firing events should be marked with @fires tag followed by event name, without class name. See example below: 

            /**
             * @expose
             * @event lynx.models.AuthModel#login
             */

            /**
             * Executes the login action.
             *
             * @expose
             * @fires login
             */
            lynx.models.AuthModel.prototype.login = function () {
                this.getCtrl().login();
            };
