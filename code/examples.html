examples in documentation

Some objects may be documented with code examples (see @example annotation). LynxSite extends @example annotation with new tag <run/>. See following example: 

    /**
     * @example <run/>
     * <input type="text" <b>lx-blur</b>="out = 'blur on element'" />
     * {{out}}
     */

It will cause JSDoc to generate both code snippet and live example build authomatically on the same code snippet provided in comment.

Annotation @example allows use of <caption> tag. You may use either of or both tags <caption> and <run/>. Order of these tags is not important. Just keep in mind to put them in separate lines as presented in following example. 

    /**
     * @example <run/> | <externalrun/>
     * <input type="text" <b>lx-blur</b>="out = 'blur on element'" />
     * {{out}}
     *
     * @example <run/> | <externalrun/>
     *          <caption>This is caption to be put in documentation</caption>
     * <input type="text" <b>lx-blur</b>="out = 'blur on element'" />
     * {{out}}
     *
     * @example <caption>This is caption to be put in documentation</caption>
     *          <run/> | <externalrun/>
     * <input type="text" <b>lx-blur</b>="out = 'blur on element'" />
     * {{out}}
     */

You may still use @example without any of tags or just with caption tag.

Usage

Mechanism similar to "Examples" is provided for "Usage" section. One can insert code into documentation annotatig it with "@lynxsiteusage" JSDoc tag. The code can have <caption> tag with some text to describe the code, but it can't be run in place via <run/> tag.

If comment contains @arg tags then resulting "parameters" section will be placed under "Usage" section. If there is no "Usage" section, "Parameters" will be top level section on it's own.

<externalrun/> tag operates like <run/> tag but also can send code to jsFiddle.
