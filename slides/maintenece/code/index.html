                Lynx is using Google Closure Compiler in Advanced Optimization mode.

    runnable without compilation for simple debugging
    
            
    
                Lynx application on developers env can be run in DEBUG mode where application is loaded
                from sources. This give as simple debugging. In DEBUG mode also external libraries are loaded from
                source.

    documented
    
            
    
                By using jsdoc we generating the documentation directly form source code. JSDoc and Google Closure Compiler
                is using the same annotations.

  • 
            
    well styled whit build time validation
    
            
    
                Lynx code is also validated by our own tool very similar to Java Checkstyle.
            
    
        
    • 
        
  • 
            
    testable whit simple coverage measure
    
            
    
                For testing we are using the Karma test runner and Jasmine.
                Karma give as also coverage report generation.
            
    
        
    • 
        
  • 
            
    modularized and organized in namespaces
    
            
    
                Lynx code is split in to few modules, but we are also strongly using namespaces for well readable
                code organization. During compilation namespaces are removed, so there is no any overload from
                performance perspective.

                During framework development we are using the JetBrains WebStorm IDE. Webstorm is well supporting the
                JsDoc/Closure Compiler annotations. This give as auto-completion and code validation direct in
                IDE.
