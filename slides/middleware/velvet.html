middleware - velvet

Velvet, being a REST based service, enables platform integration with products. Velvet is a Web Service application which sits between OSA Providers and remote internet applications. Velvet communicates with underlying OSA Provider using OSA Client Framework through trusted and high-speed (low latency) network. Velvet external API is designed to work in not trusted and high latency networks. There are could be various types of Velvet clients including mobile apps, desktop apps, web apps, portals, Mashups, modern JavaScript/AJAX or HTML5 clients, RIA Flash/Silverlight clients.

  • OData

    The Open Data Protocol (OData) is a Web protocol for querying and updating data that provides a way to unlock your data and free it from silos that exist in applications today. OData does this by applying and building upon Web technologies such as HTTP, Atom Publishing Protocol (Atom Pub) and JSON to provide access to information from a variety of applications, services, and stores. The protocol emerged from experiences implementing Atom Pub clients and servers in a variety of products over the past several years.

  • fine-grained, documented API

    Clean API to speed-up learning curve and development of client applications. Fine-grained service API is very good from reusability perspective, Velvet API is fine-grained but also solves performance issues.

  • self-diagnostics

    Velvet services provide self-diagnostics capabilities to simplify deployment, troubleshooting and monitoring.

  • secure

    Velvet service will are exposed to the Internet and secure communication is protecting user from private data leaks. There are a lot of authentication techniques. Although the Velvet supports most important authentication mechanisms.

  • compatibility

    Velvet API should be fully backward compatible, all breaking changes in OSA API are resolved on Velvet side.
