middleware - OSA

OSA (On-line Service Architecture) is a GPO asset consisting of standard API contracts and development frameworks for Java and .NET to build applications that use these API contracts. The development framework simplifies the programming model and adds common functionality like validation, monitoring & policy (‘Watchdog’), version transformation and diagnostics.

  • server to server

    OSA is designed to work in trusted and high speed networks and not intended to be exposed to the open Internet. As side effect developers have to develop OSA wrapper services for every application that needs to be connected with OSA provider though the Internet.

  • standardized

    On-line Services Architecture (OSA) sets out to provide standardized interfaces to on-line functionality
    • Simplify the process for developing services
    • Standardize service definitions in a heterogeneous landscape
    • Simplify the process for connecting to services as a client – and eliminate point-to-point implementations

  • domains

    Services are defined in domains – research, identity, user management, case management, etc. One of those domains is Research (the Global Atlas domain). The implementation of an OSA-compliant service on Global Atlas is the Atlas RSI (Research Service Interface).
