lynx framework - services

Lynx is providing stateless services layer which have to cover and simplify work with OSA/Velvet API. 

                

                

                    

                

            

        

    

    
    
        
  • 
            
    promising
    
            
    
                Services are providing data as a promisses/future objects.
                You can but you are not forced to usage of callbacks.
                If promises are binded to template, application automatically repaint the UI of promise resolving.
            
    
        
    • 
        
  • 
            
    clear api
    
            
    
                Services are created for covering the javascript parameters transformation to raw http request.
                You do not have to know the Velvet to use it. All services are "static" typed, and well documented.
            
    
        
    • 
        
  • 
            
    testable
    
            
    
                Services are covered by the unit and sytem tests.
                Durring the framework development all changes and incompatibilities
                with midellware level are quickly and simple detected.
                This provaids stability and simplifies integration.
            
    
        
    • 
        
  • 
            
    separation
    
            
    
                Developers can use services layer without any other Lynx code.
                The services layer can be use separatelly in any AngularJs application
                without any other layer usage.
            
    
        
    • 
        
  • 
            
    cross-domain
    
            
    
                Services supports full cross-domain comunication, also in IE7/8/9.
                The only thing what have to be done is exposing the Velvet instance on subdomain.
