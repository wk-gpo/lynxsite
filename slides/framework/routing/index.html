lynx framework - routing

The one of the key problem with the SPA is keeping, saving and sharing the state of application. Lynx is providing own declarative solution for routing and bookmarking. 

            

        

        

    


    
    
        
  • 

            
    segmentation
    
            
    
                lynx application can be organized in pages
            
    
        
  • 
            
    data binding
    
            
    
                By simple declaration application will automatically populating data from
                model to URL. The same declaration is used for populating data from URL to model.
            
    
        
  • 
            
    action trigering
    
            
    
                By simple declaration mached routing can trigger action, for example call method on model.
            
    
        
  • 
            
    independent
    
            
    
                Routing is declarative and independent from other application development code. Can be added
                or modified on any development step.
