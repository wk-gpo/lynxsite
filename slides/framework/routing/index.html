lynx framework - routing

The one of the key problem with the SPA is how to keep, save and share application state. Lynx provides declarative solution for routing and bookmarking. 

            

        

        

    


    
    
        
  • 
            
    segmentation
    
            
    
                lynx application can be organized in pages and components
            
    
        
  • 
            
    data binding
    
            
    
                By simple declaration application will be able to automatically populate data from
                model to URL. The same declaration is used for model updates with values provided as parts of page URL.
            
    
        
  • 
            
    action triggering
    
            
    
                By simple declaration matched routing can trigger action, for example call method on model. That
				keeps separation of code both in Lynx framework but also in your application
            
    
        
  • 
            
    independent
    
            
    
                Routing is declarative and independent from other application development code. Can be added
                or modified on any development step.
