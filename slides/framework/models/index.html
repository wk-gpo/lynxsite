lynx framework - models

Lynx provides layer of reusable and statefull models designed to keep state of the application. Models encapsulate the service calls and hide synchronization code required. 

            

            

                

            

        

        

    


    
    
        
  • 
            
    statefull
    
            
    
                Models keep the application state.
                If some data have to be reused this data have to be stored in model to limit the services calls number.
            
    
        
    • 
        
  • 
            
    global
    
            
    
                Models are globally available. You can bind your expressions to model's actions from any template.
                Models instances are created declaratively and it is possible to create many instances of the same model.
            
    
        
    • 
        
  • 
            
    self control
    
            
    
                Models care about integration of any actions. If some action require multiple services call
                model will do this by itself and will take care of data integrity.
            
    
        
    • 
        
  • 
            
    testable
    
            
    
                Models are covered by unit tests.
                Models are independent from UI what make testing very easy.
            
    
        
    • 
        
  • 
            
    integration
    
            
    
                There is possibility for cross models communication. Models may trigger events. By handling those event it is possible to 
				perform some additional actions.
