lynx framework - models

Lynx is providing reusable and stateful models layer for keeping the state of Application. Models colspad the services calls and hide the all synhronization code. 

            

            

                

            

        

        

    


    
    
        
  • 
            
    statefull
    
            
    
                Models are for keeping the applications state.
                If some data have to be reused this data have to be stored in model for limiting the services calls.
            
    
        
    • 
        
  • 
            
    global
    
            
    
                Models are globaly avaliable. You can bind your expressions to models actions from any template.
                Models instances are created declarative and you are able to create many instances of the same model.
            
    
        
    • 
        
  • 
            
    self controll
    
            
    
                Models care about integration of any actions. If some action require multiple services call
                model will do this by self, and will care of data integrity.
            
    
        
    • 
        
  • 
            
    testable
    
            
    
                Models are covered by the unit tests.
                Models are independend form UI what give as simplicity of testing.
            
    
        
    • 
        
  • 
            
    integration
    
            
    
                There is possibility for cross models comunication. Also models are triggering events, so we are able to
                handle them to perform some additional actio.
