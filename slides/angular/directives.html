angularJS - directives

Directives are a way to teach HTML new tricks. During DOM compilation directives are matched against the HTML and executed. This allows directives to register behavior, or transform the DOM. For creating the reusable UI components Lynx is using Directives. Directives covers:
  • generic simple UI controls (tree, popup, …)
  • functional UI components (login form, search form, search results, …)
  • UI layout components (split panel, …)
  • custom events handlers (click, keypress,...)
  • logical elements (if, switch, loops ....)

  • easy asynchronous

    Developers don’t have to care about asynchronous actions, do not need to use callbacks, synchronization, etc...

  • readable

    Simple, readable and natural syntax of binding expressions based on referenced API.

  • flexible

    Flexible and open model, without any inheritance/wrappers/etc requirements. Models are JavaScript objects, arrays, primitives, it doesn’t matter. Angular Js provides also easy data conversion mechanism by filters.

  • isolation

    There is no direct DOM manipulation and no DOM referencing from logic. Perfect isolation of Model and View Layer. This dramaticaly siplyfing testing.
