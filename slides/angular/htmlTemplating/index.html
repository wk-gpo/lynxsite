angularJS - html templating

Throughout application development cycle HTML remains the key artifact development work revolves. Development tasks regards to HTML decoration using Angular-specific mechanism. It allows seamless integration of UX Web designer and application developer work. 

            

            

                

            

        

        

    


    
    
        
  • 
            
    readability
    
            
    
                HTML is most popular, commonly known and natural web language/technology for
                creating web pages/applications UI. HTML is well readable. Code is self-describing - you can
                understand details from looking at the view / html source.
            
    
        
    • 
        
  • 
            
    continuous integration
    
            
    
                By using HTML as a key asset we are able to continuously integrate work of UX Designers,
                Web Designers and Application Developers.
            
    
        
    • 
        
  • 
            
    segmentation
    
            
    
                Extremely easy UI segmentation by inclusion of HTML snippets. Any snippet can be reused in application or
                very easily transformed into component. Segmentation of HTML enables lazy loading of application elements
                which increase performance.
            
    
        
    • 
        
  • 
            
    life edit
    
            
    
                By using HTML we are able to provide live application editing.
            
    
        
    • 
        
  • 
            
    stylable
    
            
    
                The styling of application can be done without any assistance from development team.
                The UX team may do by itself without any delays. Natural and simple works well with any Css and/or RWD Frameworks
                (Foundation, Bootstrap etc...)
