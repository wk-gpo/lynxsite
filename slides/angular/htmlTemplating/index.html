angularJS - html templating

Throughout development cycle application HTML remains the key artefact development work revolves. The work is about decoration of application HTML using Angular-specific mechanism. It allows seemless integration of UX Web designer and application developer work. 

                

                

                    

                

            

        

    


    
    
        
  • 
            
    readability
    
            
    
                HTML is most popular, common known and natural web language/technology for
                creating web pages/applications UI. HTML is well readable. Code reads exactly as it looks - you can
                understand details from looking at the view / html source.
            
    
        
    • 
        
  • 
            
    continuously work integration
    
            
    
                By using HTML as a central point we are able to continuously integrate work of UX Designers,
                Web Designers and Application Developers.
            
    
        
    • 
        
  • 
            
    segmentation
    
            
    
                Extremely easy UI segmentation by inclusion of html snippets. Any snippet can be reuse in application or
                simple transformed to component. Segmentation of html provides lazy loading mechanism which speedy
                application loading time and increse performance.
                By using HTML we are able to simple provide live application editing.
            
    
        
    • 
        
  • 
            
    stylable
    
            
    
                The styling of application can be done without any incorporation of Development team.
                The UX team can done it by self. Natural and simple plays well with any Css and/or RWD Frameworks
                (Fundation, Butsrtap etc...)
